楼上兄弟说的没错，Win7 扫雷会调用 SLGetWindowsInformationDWORD 来检查是否有使用权限。

差不多长这个样子：

DWORD value;
SLGetWindowsInformationDWORD(L"Shell-InBoxGames-Minesweeper-EnableGame", &value);

我们可以写一段简单的程序来验证系统是否有运行 Win7 扫雷的权限：

#include <iostream>
#include <Windows.h>
#include <slpublic.h>
#pragma comment(lib, "slc.lib")
using namespace std;

int main()
{
    DWORD value;
    SLGetWindowsInformationDWORD(L"Shell-InBoxGames-Minesweeper-EnableGame", &value);
    cout << value << endl;
    return 0;
}

这段代码在 Win7 下，value = 1；在 Win10 下，value = 0。

所以，SLGetWindowsInformationDWORD 函数做了什么？

SLGetWindowsInformationDWORD 函数会从注册表 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\ProductOptions\ProductPolicy 项中去查询权限值。

而 ProductPolicy 项是二进制值，需要解析才能获取其中的信息，可以参考这份资料：

Remko Weijnen's Blog (Remko's Blog)​www.remkoweijnen.nl/blog/2010/06/15/having-fun-with-windows-licensing/

然后我又写了一份简单的 C# 程序，用来从注册表读取权限列表。然后使用 Linq 去查询我需要的项。

源码奉上：

WinLicensing - ProductPolicy​github.com/zmjack/WinLicensing
static void Main(string[] args)
{
    var bytes = Registry.GetValue(
        @"HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\ProductOptions", 
        "ProductPolicy", null) as byte[];
    var policy = new ProductPolicy(bytes);
    var values = policy.Values.Where(x => x.Key.Contains("InBoxGames"));

    Echo.BorderTable(new[] { "Key", "Value" }, values.Select(v => new[]
    {
        v.Key,
        v.Value.Select(x => x.ToString("x2")+ " ").Join(""),
    }).ToArray(), new[] { 48, 24 });

    Console.ReadLine();
}
  • 读注册表 ProductPolicy 项值
  • 解析二进制
  • 输出包含 InBoxGames 的所有键名及其值

那么修改 ProductPolicy 是否可行？

ProductPolicy 项包含了一个权限列表用于控制一些内建项的运行，似乎不能轻易修改。

可能，如楼上所说方法，修改 EXE 的判断跳转是最简单的方式了。

2021-04-07 更新：

Win7 下的游戏会调用 slc.dll 的 SLGetWindowsInformationDWORD 方法去检查权限。

那么，是否可以创造一个自己的 slc.dll，尝试了下可行。

首先，新建一个 Win32 DLL 项目（名称为 slc）：

修改 dllmain.cpp，导出 SLGetWindowsInformationDWORD 函数：

// dllmain.cpp : Defines the entry point for the DLL application.
#include "pch.h"

BOOL APIENTRY DllMain( HMODULE hModule,
                       DWORD  ul_reason_for_call,
                       LPVOID lpReserved
                     )
{
    switch (ul_reason_for_call)
    {
    case DLL_PROCESS_ATTACH:
    case DLL_THREAD_ATTACH:
    case DLL_THREAD_DETACH:
    case DLL_PROCESS_DETACH:
        break;
    }
    return TRUE;
}

extern "C" _declspec(dllexport) HRESULT SLGetWindowsInformationDWORD(
    PCWSTR pwszValueName,
    DWORD* pdwValue
)
{
    *pdwValue = 1;
    return S_OK;
}

编译后将生成的 slc.dll 放置于游戏根目录后运行游戏即可。

我也一直纠结这个问题，直到我昨天查到了这个资料

I installed the Win7 games from Winaero – all of the downloaded files I checked, and I checked all but a few, were legit (i.e. no unwanted baggage), so downloading it and installing is probably the simplest solution. That said, if you want an alternate method to getting Win 7 Games to work, after reverse engineering what Winaero did to get them to work, here’s how you can DIY:
(1) Since the upgrade to WIN10 Anniv edition wipes out everything but the directory structure for Win 7 games, get an old copy of Win 7 games and copy it back to either c:\Program Files or c:\Program Files (x86).
(2) get an old copy of c:\Windows\System32\CardGames.dll and copy it back to c:\Windows\System32.
(3) [this is the hard part] create an override to slc.dll and put a copy in EVERY subdirectory of ..\MicrosoftGames contains a game [ex: ..\Microsoft Games\Solitaire; ..\Microsoft Games\Mahjong – this is the SAME directory as the .exe]. The reason the games no longer work is that the .exe calls slc.dll to validate the game is legit. The slc.dll in Windows\System32 returns an “invalid” response, so the game fails to launch. The workaround is to create an override to slc.dll. To do this, you have to create a new dumbed down dll that returns a “valid” return value. Here’s a assembly language version that works with MS Visual Studio:
.code
WINAPI proc hInstDLL:DWORD, fwdReason:DWORD, lpvReserved:DWORD
ret
WINAPI endp
SLGetWindowsInformationDWORD proc
ret
SLGetWindowsInformationDWORD endp
end
Note that this workaround does NOT require the hacks (changing bits in the .exe) or any other tweeks mentioned in posts to get Win 7 games working for the earlier versions of Win10. Providing a workaround version of slc.dll works with either the hacked .exes or the original .exes. Note also that the solution outlined here provides only the MINIMUM to having a working version of Win 7 games. It does not add the games to the start menu or any other “extras” that the Winaero download provides.
If you don’t want to go through the bother of creating your own slc.dll, you can image your PC, install the Winaero package, copy off slc.dll from any of the ..\Windows Games\xxx\ subdirectories, restore your PC from the image, and use Winaero’s slc.dll in place of creating your own .dll.

然后问题就解决了，那个包就OK，win7游戏
